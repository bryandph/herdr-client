crate is GitHub-primary). The umbrella / upstreaming tracker is bryan/nixspace#88
(the "workmux fork track"),

## Why

The community herdr multiplexer backend for workmux (upstream
`raine/workmux#195`, branch `GregoryTomy/workmux@herdr-backend`) drives herdr
by shelling out to the `herdr` CLI and string-parsing a JSON envelope per
operation — a process spawn per call, untyped, and silently broken by any
herdr protocol change. herdr publishes a complete JSON Schema of its socket
API (`herdr api schema --json`; bundled in-repo at
`docs/next/api/herdr-api.schema.json`, `protocol` 16, JSON Schema draft
2020-12), so the brittleness is unnecessary. A source merge of the two
codebases is rejected on licensing: herdr is AGPL-3.0, workmux is MIT (see
`raine/workmux:LICENSE`), so a merged binary is virally AGPL — whereas a
separate program speaking herdr's documented socket protocol is a clean
boundary. A spike this session confirmed the play: typify generates all five
sub-schemas with zero errors, the output compiles clean, and it round-trips to
herdr's exact wire shape.

## What Changes

- Introduce a standalone Rust crate `herdr-client` (MIT-or-Apache),
  `typify`-generated from herdr's published schema. The schema is a container
  of five self-contained sub-schemas (request, success_response,
  error_response, event, subscription_event) with zero cross-references; a
  small transform flattens each `#/schemas/<n>/$defs/X` → `#/$defs/X` into a
  root schema typify consumes.
- Generated types are **checked in** (not `build.rs` codegen), regenerated by a
  script against a herdr-version-pinned schema, so Nix builds stay
  hermetic/offline and a herdr protocol bump surfaces as a reviewable diff or
  compile break (typed drift detection is the point).
- Add a thin hand-written veneer over the generated types: a socket transport
  (`$HERDR_SOCKET_PATH`, newline-delimited JSON request/response, typed errors)
  plus typed method fns. Required because typify renders the top-level
  `Request` `oneOf` as an untagged `Variant0..N` enum (const-discriminated on
  `method`); the veneer consumes the params/result structs directly. The event
  subscription stream is out of scope (v2).
- Rewire the workmux fork's `src/multiplexer/herdr.rs` to depend on
  `herdr-client` instead of shelling out — same `Multiplexer` trait surface,
  typed internals. Fix the `#195` bug where deferred cleanup hardcodes `herdr`
  instead of honoring `$HERDR_BIN_PATH`.
- Repoint nixspace's `workmux` flake input from `github:raine/workmux` to the
  fork carrying the typed backend; add `herdr-client` as a flake input the fork
  consumes.

## Capabilities

### New Capabilities

- `herdr-workmux-backend`: workmux drives herdr as its multiplexer through a
  typed, schema-generated client pinned to a herdr protocol version (fail-loud
  on mismatch), delivered to the fleet via pinned fork + crate flake inputs,
  with generated types checked in and drift-detected.

## Impact

- **This repo** (`bryan/herdr-client`, git.bph; GitHub mirror later) hosts the
  `herdr-client` crate and this change's design of record. A separate **new
  external repo** (mirrored git.bph + GitHub) carries the `workmux` fork. This
  change owns their design of record and the nixspace-side wiring.
- **nixspace `flake.nix`**: the `workmux` input url changes (currently
  `github:raine/workmux`, `flake.nix:400`); a new `herdr-client` input is added
  and `follows`-deduplicated per `mem:style/nix-follows-discipline`.
- **No behavior change** to `home/modules/shell/{workmux,herdr}.nix` or the
  vendored HM modules beyond the `pkgs.workmux` package source.
- **Upstreaming** (tracked on #88, not blocking this change): candidate 2
  (Codex `PermissionRequest` → waiting) stays a standalone PR; the three
  generic `Multiplexer` trait methods from `#195` (atomic worktree/workspace
  lifecycle) are upstreamable to `raine/workmux` regardless; `herdr-client` is
  offered to herdr as an official Rust client.
- **Reuse**: the typed client is consumable by the mandala Rust rewrite
  (#79/#95) if its ratatui TUI ever drives herdr panes.

## Non-goals

- The broader workmux+herdr **unification** — herdr owning worktree creation
  vs workmux (decision 1), the container/sandbox isolation tier (decision 2),
  single-channel status reconciliation, and OpenSpec pre-merge gating — remains
  an open, separately-tracked design. This change is only the typed client +
  backend rewire.
- No source merge of the two codebases (AGPL boundary, above).
- No change to herdr's own agent detection: it already handles nix-wrapped
  binaries (`.claude-wrapped` / `.opencode-wrapp`, with a NixOS-path regression
  test), which is why #88 candidate 3 (workmux-side wrapper detection) is
  dropped rather than carried.
- The herdr event/subscription stream and any herdr-native worktree UI
  integration are deferred (v2).
